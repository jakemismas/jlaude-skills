## SOW Structure (Industry Standard)

### 1. Project Overview
- Project name and client name
- Brief description (2-3 sentences)
- Business objectives and success criteria
- Reference to governing MSA or proposal

### 2. Scope of Work
Organized by phase. Each phase includes:
- Phase name and description
- Specific activities (numbered)
- Deliverables (with acceptance criteria)
- Duration estimate
- Dependencies on prior phases

### 3. Out of Scope
Explicit exclusions. Be specific. Vague exclusions create disputes.

### 4. Assumptions and Dependencies
- Technical assumptions (org edition, licenses, API access)
- Organizational assumptions (stakeholder availability, decision timeline)
- Resource assumptions (client admin availability, data quality)
- External dependencies (third-party integrations, vendor timelines)

### 5. Client Responsibilities
What the client must provide and by when. Be specific about:
- Access (org credentials, VPN, sandbox provisioning)
- Resources (subject matter experts, admin, data steward)
- Decisions (timely feedback, approval cycles)
- Data (clean data, migration source files)

### 6. Timeline
Phase-based with milestones. Include:
- Start and end dates per phase
- Key milestones and decision points
- Buffer for client review cycles

### 7. Investment / Payment Terms
[JAKE REVIEW REQUIRED for all pricing]
- Fixed fee, T and M, or blended model
- Payment schedule tied to milestones or monthly
- Expense policy

### 8. Change Order Process
How scope changes are handled:
- Written change request required
- Impact assessment (timeline, cost)
- Mutual written approval before work begins

## Quality Standards
- Every deliverable has acceptance criteria
- Every assumption has a consequence if invalid
- No ambiguous language ("reasonable," "as needed," "best efforts")
- All dates are absolute, never relative
